# 教师侧小组任务完成度流程说明

更新时间：2026-04-02  
状态：已实现并完成后端冒烟测试、前端构建验证

## 1. 目标

补齐教师侧“小组协作闭环”的下一步能力，让教师在工作台中直接查看：

1. 当前班级在当前学案下有哪些“小组共同提交”任务。
2. 每个任务有多少小组未开始、已提交、已评阅。
3. 每个小组在整份学案中的总体完成度。
4. 某个小组由谁提交、提交了多少文件、是否已评分、是否被推荐。

## 2. 页面入口

教师或管理员登录后进入：

1. `/login/staff`
2. `/staff/dashboard`

进入教师工作台后：

1. 页面顶部先选择班级和学案。
2. 在“小组任务进度”卡片中查看当前组合下的小组共同提交完成度。
3. 进度区域会随班级、学案切换自动刷新。
4. 教师可直接点击“打开评分页”跳转到该任务的评分页面。

## 3. 前端实现

前端文件：

1. `apps/web/src/modules/staff-dashboard/StaffDashboardPage.vue`

本次新增了两层信息：

1. 小组总览进度卡：
   - 按小组汇总已提交任务数、已评阅任务数、待完成任务数。
   - 显示每个小组在当前学案中的提交完成度和评阅进度。
2. 按任务展开的明细卡：
   - 显示该任务下每个小组的状态：`未开始 / 已提交 / 已评阅`
   - 显示提交人、更新时间、文件数、得分、推荐状态
   - 支持直接跳转到教师评分页

## 4. 后端接口

后端文件：

1. `apps/api/app/api/v1/endpoints/staff.py`

接口：

`GET /api/v1/staff/classes/{class_id}/plans/{plan_id}/group-task-progress`

用途：

1. 教师按班级 + 学案查看小组共同提交任务进度。
2. 只统计 `submission_scope = group` 且非阅读任务。
3. 返回班级摘要、学案摘要、全局统计、按任务展开的小组进度项。

返回重点：

1. `summary.group_count`
2. `summary.task_count`
3. `summary.submitted_count`
4. `summary.reviewed_count`
5. `tasks[].items[].status`
6. `tasks[].items[].submitted_by_name`
7. `tasks[].items[].score`
8. `tasks[].items[].is_recommended`

## 5. 权限规则

1. 教师只能查看自己可访问班级的小组任务进度。
2. 管理员可查看任意班级。
3. 非负责教师访问他班数据时返回 `403`。

## 6. 测试覆盖

后端测试文件：

1. `apps/api/tests/test_smoke.py`

本次新增覆盖场景：

1. 教师读取某班某学案的小组任务进度。
2. 学生以小组模式提交任务后，教师能看到对应小组变为“已提交”。
3. 教师评分后，状态会更新为“已评阅”并带回分数。
4. 非负责教师访问该班进度接口会返回 `403`。
