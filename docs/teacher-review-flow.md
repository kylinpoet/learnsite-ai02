# 教师作品评分流程说明

更新时间：2026-03-31  
状态：ACTIVE  
适用范围：教师作品总览页、任务评分页、大窗口评分工作台、等级评分、批量评分、评分模板云端同步、评分模板分组排序、附件预览下载、附件批量下载、本地演示数据与联调流程

## 1. 本轮完成内容

当前教师端作品评分闭环已经覆盖以下能力：

1. 教师可查看“已有学生提交”的任务总览。
2. 教师可在总览页切换“仅看未评阅任务”，并按课程、课次快速筛选任务。
3. 教师可进入单个任务的评分页查看学生提交列表。
4. 教师可在任务页切换“仅看未评阅作品”，并按班级筛选学生提交。
5. 教师可查看学生富文本作品说明和附件元数据。
6. 教师可使用 `G / A / B / C / D / E / F` 七档等级评分，分别对应 `120 / 100 / 80 / 60 / 40 / 20 / 0`。
7. 教师点击等级分按钮后，系统会立即自动保存评分结果，并自动进入下一位学生继续评分。
8. 当“仅看未评阅作品”开启且教师刚好完成最后一份待评作品时，系统会自动切换为“显示全部作品”，并继续停留在当前作品上，方便教师确认评分结果。
9. 教师可勾选多份作品进行批量评分。
10. 教师可维护个人评分模板，并支持云端同步、分组展示、组内排序、“套用模板 / 仅插入评语 / 保存当前为模板”。
11. 教师可在大窗口评分工作台中直接展示图片、文字、PDF 类附件。
12. 教师可预览支持在线查看的附件。
13. 教师可下载所有附件。
14. 教师可将左侧已勾选作品的全部附件一次性打包为 ZIP 下载。
15. 作品一旦被教师评分，学生端自动禁止再次提交。
16. 本地演示环境启动后会自动恢复标准演示数据，并补齐演示附件文件，避免反复测试导致数据变脏。

## 2. 页面说明

### 2.1 教师作品总览页

文件：`apps/web/src/modules/staff-submission/SubmissionOverviewPage.vue`  
路由：`/staff/submissions`

页面职责：

1. 展示任务数、作品总数、已评阅数、待评阅数、平均分。
2. 按任务维度展示课程、任务、课次、提交数、已评阅、待评阅、平均分、最近提交时间。
3. 默认开启“仅看未评阅任务”，帮助教师先处理还有待办的任务。
4. 教师可进一步按课程、课次缩小范围，快速定位某一单元或某一课的任务。
5. 教师可一键“恢复默认”，回到仅看待评任务的默认视图。
6. 点击任一任务进入该任务评分页。

### 2.2 教师任务评分页

文件：`apps/web/src/modules/staff-submission/SubmissionTaskPage.vue`  
路由：`/staff/submissions/:taskId`

本轮补充：
1. 任务评分页新增“教师端推荐作品展示”区域。
2. 当教师将作品评为 `G`，且分数为 `120` 时，该作品会自动进入当前任务的推荐展示。
3. 推荐作品在学生当前任务页也会同步展示，形成教师评分到学生展示的自动闭环。
4. 如后续把 `G` 改成其他等级，推荐状态会随评分结果自动取消。

页面分为三部分：

1. 顶部任务信息区
   - 任务名称
   - 所属课程
   - 所属单元
   - 所属课次

2. 中部统计区
   - 提交总数
   - 已评阅
   - 待评阅
   - 平均分

3. 下方双栏工作区
   - 左侧：学生提交列表、未评阅筛选、班级筛选、批量评分、批量模板、批量附件下载
   - 右侧：单个作品评分面板、附件预览下载入口、单份模板、大窗口评分入口

补充说明：

1. 单份评分和批量评分共用同一套模板数据。
2. 模板按教师账号保存到后端数据库，支持同账号跨设备同步。
3. 前端保留旧版本地模板迁移逻辑，首次进入页面时会自动尝试合并到云端。
4. 模板在下拉选择器、快捷插入区、模板管理弹窗中都会按分组显示，组内按排序值升序展示，并支持上移 / 下移调整。
5. 单份评分与批量评分默认使用七档等级评分；系统内部仍保存为数值分数。
6. 教师可进入“大窗口评分工作台”，直接查看图片、文字、PDF，并在点击等级后立即保存并切换到下一份作品。
7. 当“仅看未评阅作品”筛选下只剩最后一份待评作品时，保存后系统会自动切换为“显示全部作品”，避免评分面板瞬间变为空状态。

## 3. 教师操作流

### 3.1 先看待办任务

推荐路径：

1. 登录教师账号。
2. 进入“作品评分”。
3. 保持“仅看未评阅任务”为开启状态。
4. 如需聚焦某门课程或某一课次，可继续选择课程、课次筛选。
5. 点击任一待处理任务进入评分页。

### 3.2 课堂快速批改

适用场景：多名学生完成质量接近，希望统一给出基础评分或通用反馈。

操作方式：

1. 如班级较多，先在任务页左侧使用“班级筛选”定位目标班级。
2. 在任务页左侧表格勾选多位学生。
3. 在“批量评分”区域选择统一等级分或填写统一评语。
4. 点击“批量保存评分”。
5. 系统刷新列表和统计，已评阅作品将不再出现在“仅看未评阅作品”筛选结果中。

### 3.3 单个精细批改

适用场景：需要逐一查看说明、附件并单独给出反馈。

操作方式：

1. 点击某位学生作品，右侧加载该学生详情。
2. 如需切换其他班级，可先调整“班级筛选”。
3. 查看学生提交的富文本作品说明。
4. 预览或下载附件。
5. 如需写个性化反馈，可先填写教师评语。
6. 点击任一教师等级分按钮后，系统会立即保存当前评分与评语，并自动进入下一位学生。
7. 如果只修改评语、不调整分数，可再使用下方“保存当前修改”按钮手动保存。

### 3.4 使用大窗口评分工作台

适用场景：需要放大查看学生作品，边看边评，完成当前评分后连续处理下一位学生。

操作方式：

1. 在学生提交列表点击“进入评分”，或在右侧面板点击“大窗口评分”。
2. 进入全屏评分工作台后，系统会自动加载当前学生作品。
3. 如附件为图片、文字、PDF，系统会直接在主预览区展示内容。
4. 如学生有多个附件，可点击文件卡片切换当前展示内容。
5. 如需写个性化反馈，可先在右侧填写评语。
6. 点击 `G / A / B / C / D / E / F` 中任一等级后，系统会立即保存当前评分与评语，并自动定位到下一位学生作品继续展示。
7. 如果这是当前“仅看未评阅作品”筛选下的最后一份待评作品，系统会自动改为显示全部作品，并继续停留在刚刚保存的作品上。
8. 当前筛选列表全部评分完成且没有可继续展示的作品时，工作台才会自动结束连续评分流程。

### 3.5 使用评分模板与快捷评语

适用场景：教师希望反复复用常用分数与评语，减少重复输入。

操作方式：

1. 在“批量评分”区或“单个评分面板”中选择一个评分模板。
2. 点击“套用模板”，系统会带入模板中的推荐等级分和模板评语。
3. 点击“仅插入评语”，系统只会把模板评语追加到当前评语框中，不覆盖已有分数。
4. 如当前填写的等级分和评语值得复用，可点击“保存当前为模板”。
5. 在“评分模板管理”弹窗中，可新建、编辑、删除模板，也可为模板设置分组、排序值，并通过“上移 / 下移”调整组内顺序；还可直接套用到单份或批量评分。
6. 如教师此前在旧版本浏览器中维护过本地模板，系统会首次自动迁移到当前账号的云端模板列表。

## 4. 附件预览与下载

### 4.1 当前支持的在线预览类型

系统当前支持在线预览以下类型：

1. PDF
2. PNG / JPG / JPEG / GIF / WEBP / SVG
3. TXT
4. Markdown

说明：

1. 预览采用受权限保护的后端接口。
2. 前端通过带认证头的二进制请求读取文件，再在浏览器中展示。
3. 不支持在线预览的文件类型仍可直接下载。

### 4.2 当前下载策略

所有附件均支持下载，包括：

1. 图片
2. PDF
3. DOCX
4. Markdown
5. 其他普通附件

补充说明：

1. 单个作品仍支持逐个附件下载。
2. 教师在任务评分页左侧勾选多份作品后，可直接点击“打包下载附件”。
3. 系统会将所选作品的全部附件打成一个 ZIP 包，并按“任务 / 学号_姓名_提交ID / 原文件名”组织目录。
4. 如所选作品都没有附件，后端会直接返回错误提示，不生成空压缩包。

### 4.3 权限规则

补充：
1. 学生仍然不能直接访问其他同学的普通作品附件。
2. 只有已经进入 `recommended_showcase` 的推荐作品附件，才允许其他已登录学生查看。

附件访问遵循以下规则：

1. 教师和管理员可访问所有作品附件。
2. 学生只能访问自己的作品附件。
3. 学生无权访问其他学生附件。
4. 附件数据库记录存在但磁盘文件不存在时，接口返回 `404`。

## 5. 后端接口

### 5.1 教师作品总览

接口：`GET /api/v1/submissions/teacher`

权限：教师 / 管理员

返回内容：

1. 汇总统计：
   - `task_count`
   - `submission_count`
   - `reviewed_count`
   - `pending_count`
   - `average_score`
2. 任务列表：
   - `task_id`
   - `task_title`
   - `course_title`
   - `lesson_title`
   - `unit_title`
   - `submission_count`
   - `reviewed_count`
   - `pending_count`
   - `average_score`
   - `latest_submitted_at`

### 5.2 单个任务评分页数据

接口：`GET /api/v1/submissions/teacher/task/{task_id}`

新增返回：
1. `summary.recommended_count`
2. `items[].is_recommended`
3. `recommended_showcase.count`
4. `recommended_showcase.items`

权限：教师 / 管理员

返回内容：

1. 任务基础信息。
2. 所属课程信息。
3. 提交统计信息。
4. 学生提交列表。

每条学生提交包含：

1. `submission_id`
2. `student_name`
3. `student_no`
4. `class_name`
5. `status`
6. `score`
7. `submitted_at`
8. `updated_at`
9. `submission_note`（当前允许富文本 HTML）
10. `teacher_comment`
11. `files`

每个附件对象包含：

1. `id`
2. `name`
3. `ext`
4. `size_kb`
5. `role`
6. `mime_type`
7. `previewable`

### 5.3 单个作品评分

接口：`POST /api/v1/submissions/{submission_id}/score`

权限：教师 / 管理员

请求体示例：

```json
{
  "score": 120,
  "teacher_comment": "Good structure. Add one more example."
}
```

规则：

1. 分数范围为 `0-120`。
2. 分数和教师评语至少填写一项。
3. 保存后作品状态自动改为 `reviewed`。
4. 已评阅作品允许教师再次修改分数和评语。
5. 当前前端默认使用等级评分，映射关系为：`G=120`、`A=100`、`B=80`、`C=60`、`D=40`、`E=20`、`F=0`。

### 5.4 批量评分

接口：`POST /api/v1/submissions/batch-score`

权限：教师 / 管理员

请求体示例：

```json
{
  "submission_ids": [6611, 6612],
  "score": 100,
  "teacher_comment": "课堂任务完成度较高，再补充一个生活中的实际案例。"
}
```

返回体核心字段：

1. `updated_count`
2. `submission_ids`
3. `task_ids`

规则：

1. `submission_ids` 至少包含 1 个作品 ID。
2. 分数和教师评语至少填写一项。
3. 批量评分会逐条更新作品状态为 `reviewed`。
4. 如果请求中存在不存在的作品 ID，接口返回 `404`。
5. 当前前端入口主要用于同一任务下的多份作品统一评分，但接口本身支持跨任务处理。
6. 当前前端默认同样使用等级评分，最终仍以数值分数写入数据库。

### 5.5 评分模板接口

接口：

1. `GET /api/v1/submissions/review-templates`
2. `POST /api/v1/submissions/review-templates`
3. `PUT /api/v1/submissions/review-templates/{template_id}`
4. `DELETE /api/v1/submissions/review-templates/{template_id}`

权限：教师 / 管理员

模板字段：

1. `id`
2. `title`
3. `group_name`
4. `sort_order`
5. `score`
6. `comment`
7. `updated_at`

规则：

1. 模板按当前教师账号隔离存储，不同教师互不可见。
2. 模板名称不能为空，长度不超过 30 个字符。
3. `group_name` 可为空，空值表示“未分组”；非空时长度不超过 40 个字符。
4. `sort_order` 可为空；为空时，后端会自动将模板追加到当前分组末尾。
5. 模板列表接口默认按“未分组优先、再按分组名称、组内排序值、更新时间倒序”返回。
6. 模板至少需要包含推荐分数或模板评语中的一项。
7. 推荐分数范围为 `0-120`。
8. 当前演示环境启动后，会为教师和管理员账号写入一组默认模板，默认包含“课堂表现”“改进建议”等分组。
9. 应用启动时会检查旧版 SQLite 中 `review_templates` 表结构，若缺少 `group_name` 或 `sort_order` 列，会自动补齐并回填历史数据。

### 5.6 附件访问接口

接口：`GET /api/v1/submissions/files/{file_id}`

权限：教师 / 管理员 / 文件所属学生

查询参数：

1. `disposition=inline`
   - 用于在线预览。
2. `disposition=attachment`
   - 用于下载。

接口行为：

1. 校验文件是否存在。
2. 校验当前用户是否有权限访问该附件。
3. 根据文件名推断 `Content-Type`。
4. 根据 `disposition` 返回内联展示或下载响应。

### 5.7 批量附件下载接口

接口：`POST /api/v1/submissions/files/batch-download`

权限：教师 / 管理员

请求体示例：

```json
{
  "submission_ids": [6611, 6612]
}
```

响应行为：

1. 接口直接返回 ZIP 二进制流，而不是 JSON。
2. 响应头会带 `Content-Disposition: attachment`，前端收到后直接触发浏览器下载。
3. ZIP 文件名默认按任务信息与作品数量生成。

规则：

1. `submission_ids` 至少包含 1 个作品 ID。
2. 接口会自动去重，避免同一作品被重复打包。
3. 如请求中存在不存在的作品 ID，接口返回 `404`。
4. 如所选作品都没有附件，接口返回 `400`。
5. 如任一附件记录存在但磁盘文件缺失，接口返回 `404`。
6. ZIP 内部会按任务和学生信息创建目录，避免不同学生上传同名文件时相互覆盖。

1. 为演示 PDF 生成可预览的最小 PDF 文件。

2. 保证自动化测试可以稳定通过。
3. 避免数据库里只有附件记录但没有实际文件。

## 7. 学生端联动规则

教师保存评分后，学生端会自动出现以下变化：

1. 学生作品详情页状态切换为已完成教师评价。
2. `can_resubmit` 变为 `false`。
3. 学生任务页提交通道被锁定。
4. 学生再次提交同一任务时会收到 `409` 错误。

当前后端返回的冲突提示为：

```text
作业已评价，不能再次提交
```

说明：

1. 前端页面中的状态文案当前使用“已评阅 / 待评阅”。
2. 后端冲突提示仍使用“已评价”。
3. 这是文案层面的轻微不一致，不影响功能，可在后续统一。
