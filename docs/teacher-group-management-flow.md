# 教师侧小组管理流程说明

更新时间：2026-04-02
状态：已实现并已通过后端测试、前端类型检查、生产构建验证

## 1. 目标

本功能用于补齐教师在课堂中的“小组管理闭环”，让教师无需离开工作台，就能完成以下操作：

1. 按班查看当前小组分布。
2. 新增空小组。
3. 按目标组数一键重组。
4. 手动调整学生所在小组。
5. 调整每个小组的组长。
6. 删除空组。
7. 让教师工作台、学生小组页、小组共享网盘同时看到最新结果。

这次实现的定位不是单独做一个复杂后台，而是在教师工作台里直接提供“分组维护入口”，优先服务真实上课场景。

## 2. 用户入口

教师或管理员登录后进入：

1. `/login/staff`
2. `/staff/dashboard`

进入教师工作台后：

1. 页面顶部先选择班级和学案。
2. 中部“小组协作”区域会自动显示当前班级的小组概览。
3. 点击“小组维护”按钮后，右侧抽屉打开。
4. 抽屉中即可完成重组、组长调整、成员调整、空组删除和保存。

当前设计没有额外跳转新页面，原因如下：

1. 教师在上课中频繁切页成本高。
2. 小组管理与课堂聚焦信息强相关。
3. 班级切换后，小组数据需要和座位图、签到状态同步变化。

## 3. 页面行为

前端文件：

1. `apps/web/src/modules/staff-dashboard/StaffDashboardPage.vue`

页面主要包含两层能力。

### 3.1 小组概览区

教师工作台中部“小组协作”卡片负责展示：

1. 当前班级的小组数量。
2. 当前班级的小组共享文件总数。
3. 每个小组的组名、组号、组长。
4. 每个小组的签到人数、未签到人数。
5. 每位成员的学号、座位、机房、签到时间。
6. 每个小组共享空间的容量、文件数、最近文件。
7. 教师直接下载共享文件。

### 3.2 分组维护抽屉

点击“小组维护”后打开抽屉，抽屉内分为两个区域。

左侧“组配置”区域：

1. 修改组名。
2. 修改组说明。
3. 选择组长。
4. 查看该组成员预览。
5. 删除空组。

右侧“成员分配”区域：

1. 查看全班学生列表。
2. 查看学生当前所在组。
3. 修改学生目标小组。
4. 查看该学生是否为当前组组长。
5. 根据签到状态、座位信息辅助教师快速判断。

顶部工具区支持：

1. 输入目标小组数。
2. 一键重组。
3. 新增小组。
4. 保存当前分组结果。

抽屉内还提供“追溯记录”面板，支持：

1. 按小组筛选日志。
2. 按事件类型筛选日志。
3. 按操作人筛选日志。
4. 按关键字搜索标题、任务、文件与说明。
5. 导出当前筛选结果为 CSV。

## 4. 操作规则

### 4.1 一键重组

教师输入目标组数后，点击“一键重组”。

系统行为：

1. 按当前班级全部学生重新平均分配。
2. 学号顺序作为稳定排序基础。
3. 每组第一个学生自动设为组长。
4. 组数不足时自动补新组。
5. 多余的小组保留，但会被清空成员。
6. 已存在的小组共享空间不会被自动删除。

这样做的目的，是避免教师一键重组时误删已有共享文件。

### 4.2 手动调组

教师可直接在成员表里为某位学生选择目标小组。

系统行为：

1. 学生会从原组移出。
2. 学生加入新组。
3. 原组若失去原组长，系统会自动补一个组长。
4. 新组若还没有合法组长，系统会自动补一个组长。

### 4.3 组长调整

教师可在每个小组的“组长”下拉框中选择新组长。

规则：

1. 组长必须是本组成员。
2. 如果教师选中的组长已不在本组，保存时会自动回退为本组第一位成员。
3. 空组没有组长。

### 4.4 新增小组

教师点击“新增小组”后：

1. 系统在当前班级下创建一个空组。
2. 自动分配新的 `group_no`。
3. 自动生成默认组名。
4. 自动准备该组未来对应的小组共享空间命名。

### 4.5 删除空组

删除操作有严格限制，只允许删除同时满足以下条件的小组：

1. 没有任何成员。
2. 没有任何共享文件。

这样做是为了避免误删已有协作成果。

## 5. 后端接口

后端文件：

1. `apps/api/app/api/v1/endpoints/staff.py`

### 5.1 获取班级小组概览

接口：

`GET /api/v1/staff/classes/{class_id}/groups`

用途：

1. 教师工作台按班展示小组与共享文件。
2. 班级切换后刷新概览区。

返回重点：

1. 班级基础信息。
2. 小组数量。
3. 共享文件总数。
4. 每个小组成员与共享空间摘要。

### 5.2 获取分组维护数据

接口：

`GET /api/v1/staff/classes/{class_id}/group-management`

用途：

1. 打开“小组维护”抽屉时初始化数据。
2. 保存后重新回填最新状态。

返回重点：

1. 班级基础信息。
2. 当前全部小组。
3. 当前全部学生。
4. 学生当前所在组、当前角色、签到状态、座位信息。

### 5.3 新增小组

接口：

`POST /api/v1/staff/classes/{class_id}/groups`

请求体：

```json
{
  "name": "可选，自定义组名",
  "description": "可选，组说明"
}
```

当前前端默认传空对象，由后端自动生成默认组名与默认说明。

### 5.4 一键重组

接口：

`POST /api/v1/staff/classes/{class_id}/groups/rebuild`

请求体：

```json
{
  "group_count": 3
}
```

校验规则：

1. `group_count` 不能小于 1。
2. `group_count` 不能大于当前班级学生数。

### 5.5 保存分组结果

接口：

`PUT /api/v1/staff/classes/{class_id}/group-management`

请求体结构：

```json
{
  "groups": [
    {
      "id": 12,
      "name": "701 Project Group",
      "description": "Teacher-adjusted project group for classroom collaboration.",
      "leader_user_id": 201,
      "member_user_ids": [201, 202, 203]
    }
  ]
}
```

校验规则：

1. 小组必须属于当前班级。
2. 小组名称不能为空。
3. 同一个学生不能在同一组内重复出现。
4. 同一个学生不能同时出现在多个组中。
5. 组长必须属于该组成员。

保存行为：

1. 先清空当前班全部小组成员关系。
2. 再按提交结果重建成员关系。
3. 同步更新组长。
4. 同步小组共享空间显示名。

### 5.6 删除空组

接口：

`DELETE /api/v1/staff/groups/{group_id}`

限制：

1. 当前教师必须有该班级访问权限。
2. 小组内不能有成员。
3. 小组共享空间内不能有文件。

删除成功后，接口会直接返回当前班的最新管理数据，前端无需再做额外拼装。

### 5.7 教师直接上传共享资料

接口：

`POST /api/v1/staff/groups/{group_id}/drive/files`

请求格式：

`multipart/form-data`

字段：

1. `files`

处理规则：

1. 仅教师或管理员可调用。
2. 当前教师必须拥有目标小组所属班级的访问权限。
3. 支持一次上传多个文件。
4. 上传成功后，接口会直接返回当前班级最新的小组概览数据，前端可即时刷新。
5. 每个文件都会写入持久化操作日志，事件类型为 `teacher_group_file_uploaded`。

### 5.8 按条件筛选小组操作日志

接口：

`GET /api/v1/staff/classes/{class_id}/group-operation-logs`

查询参数：

1. `group_id`
2. `event_type`
3. `actor_user_id`
4. `keyword`
5. `limit`

用途：

1. 教师在“小组维护”抽屉中按条件查看当前班级的追溯记录。
2. 支持快速聚焦某个小组、某位成员或某类事件。

返回重点：

1. `items`
2. `total_count`
3. `limit`

### 5.9 导出小组操作日志

接口：

`GET /api/v1/staff/classes/{class_id}/group-operation-logs/export`

查询参数：

1. `group_id`
2. `event_type`
3. `actor_user_id`
4. `keyword`

处理规则：

1. 复用与筛选接口一致的过滤条件。
2. 导出结果为带 UTF-8 BOM 的 CSV，便于 Excel 直接打开中文内容。
3. 文件名包含班级名与导出日期。

## 6. 权限规则

适用角色：

1. 教师
2. 管理员

权限边界：

1. 教师只能管理自己绑定班级的小组。
2. 管理员可以管理任意班级。
3. 非负责教师访问他班数据时返回 `403`。

## 7. 数据联动

这次小组管理不是孤立功能，它会联动以下模块：

1. 教师工作台小组概览。
2. 学生 `/student/groups` 小组页。
3. 学生 `/student/drive?tab=group` 小组共享网盘。
4. 教师侧小组共享文件查看与下载。

联动结果：

1. 教师保存分组后，学生重新进入小组页会看到最新组别。
2. 教师调整组长后，学生端会同步显示新的组长身份。
3. 教师改组名后，小组共享空间显示名会同步更新。
4. 已存在的小组共享文件不会在重组时被自动删除。
5. 教师上传讲义或素材后，学生侧小组网盘与小组页会即时看到新增文件与日志记录。
