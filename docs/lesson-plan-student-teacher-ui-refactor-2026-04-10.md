# 学案与任务交互改造任务单（2026-04-10）

状态：COMPLETED（2026-04-10）  
执行顺序：严格按 1 → 6  
目标：修复“学案导读/任务切换、教师编辑页分离、任务提交入口缺失、信息层级与视觉分区不足”的关键体验问题。

## 0. 背景共识

1. 学生端学案学习页应以“切换”而不是“上下滚动堆叠”为主。
2. 教师端学案编辑应是独立工作页，不和列表混在同一屏。
3. 网页任务与讨论任务都属于“可被教师评价的任务”，学生端必须有正式提交入口。
4. 列表信息需要更强视觉分层，文本不应连成一片。

## 1. 学生页：学案导读与任务清单改为标签页

需求：
1. `/student/courses/:courseId` 页面将“学案导读”和“任务清单”改为 `Tabs`。
2. 默认进入“学案导读”标签，可切换到“任务清单”。
3. 保留原任务卡点击进入任务能力。

验收标准：
1. 导读与任务不再上下连续堆叠。
2. 任务数量为 0 时仍能正确显示空状态。
3. 移动端标签栏可用，不遮挡内容。

状态：`已实现（2026-04-10）`

## 2. 教师学案配置：按“学案导读、任务1、任务2...”标签页呈现

需求：
1. 学案编辑区顶部改为统一标签体系：
   - `学案导读`
   - `任务 1`
   - `任务 2`
   - ...
2. 切换任务标签时，展示对应任务的完整配置卡片。
3. 保留新增任务、模板库、复制/删除/移动等能力。

验收标准：
1. 教师不再先点“任务配置”再点内部任务标签。
2. 新增任务后，能立即出现新的任务标签并可编辑。
3. 保存后任务顺序与标签顺序一致。

状态：`已实现（2026-04-10）`

## 3. 网页任务：补齐学生正式提交入口

需求：
1. 在网页任务页面保留网页 iframe 展示。
2. 同时补充“提交说明 + 附件上传 + 提交按钮”区域。
3. 使用现有 `/tasks/{id}/submit` 提交链路。

验收标准：
1. 网页任务可直接提交并落库为 `current_submission`。
2. 已提交后可再次提交覆盖（教师未评前）。
3. 教师评分页能看到网页任务学生提交内容。

状态：`已实现（2026-04-10）`

## 4. 讨论任务：补齐学生正式提交入口

需求：
1. 保留讨论区“发布回复”能力。
2. 额外补充“任务正式提交”区域（提交说明、附件、提交按钮）。
3. 讨论互动与正式提交互不冲突。

验收标准：
1. 讨论任务能发讨论，也能正式提交任务成果。
2. 教师评分页可看到讨论任务的正式提交记录。
3. 提交规则与其他任务一致（已评后不可再提）。

状态：`已实现（2026-04-10）`

## 5. 教师端：学案清单与学案编辑分离为独立页面

需求：
1. `/staff/lesson-plans` 仅保留学案清单视图（列表页）。
2. 学案编辑改为独立页（例如：`/staff/lesson-plans/new`、`/staff/lesson-plans/:planId`）。
3. 列表点击编辑后跳转独立编辑页，不再在列表下方展开编辑器。

验收标准：
1. 学案清单页不出现大体量编辑器。
2. 编辑页可完整创建/编辑/保存/发布。
3. 从编辑页可返回清单页，且列表状态可刷新。

状态：`已实现（2026-04-10）`

## 6. 视觉分区增强 + 学案缩略图

需求：
1. 优先在教师“学案清单”增强视觉分区（标题块、课次信息块、状态块）。
2. 若学案导读内存在图片，提取第一张作为列表缩略图。
3. 无图片时显示统一占位缩略图。

验收标准：
1. 学案列表信息密度降低，扫读更快。
2. 有图学案出现缩略图，无图学案有占位图。
3. 标题、课次、状态、发布时间等信息层级清晰。

状态：`已实现（2026-04-10）`
