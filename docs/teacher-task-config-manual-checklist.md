# 教师端任务配置全流程手测清单

更新时间：2026-04-09  
状态：ACTIVE  
适用范围：`/staff/lesson-plans` 页面中的“任务配置”全链路（含 CKEditor 双模式、任务模板、网页/数据提交任务资源、AI 生成弹窗、预览反馈）

## 1. 测试目标

1. 验证教师端任务配置从“新建任务”到“保存学案/发布学案”的核心链路可用。
2. 验证学案正文与任务配置中的编辑框已统一为 CKEditor，且可视化与源码模式可互相解析。
3. 验证网页任务与数据提交任务的资源上传、源码保存、预览与错误提示链路可用。
4. 验证任务模板“保存、覆盖、筛选、排序、批量管理、复用”链路可用。

## 2. 执行前准备

1. 使用教师账号登录，确认账号有课程和课次可绑定。
2. 准备至少 1 个可绑定课次（新建学案时必须选择）。
3. 准备本地测试文件：
   - `index.html`（可正常渲染）
   - `index.html + css/js` 的文件夹
   - 一个“错误入口路径”场景文件（如入口填 `main.html`，实际只有 `index.html`）
4. 若验证 AI 生成功能，需确认 AI Provider 可用且教师账号已登录。

## 3. 结果记录方式

1. 每条用例执行后记录 `通过/失败/阻塞`。
2. 失败用例记录缺陷编号、截图、重现步骤、浏览器信息。
3. 建议执行环境：
   - Chrome 最新稳定版（主验收）
   - Edge 最新稳定版（回归抽检）
